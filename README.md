# FraudDetection

A UML Profile for AI-Based Financial Fraud Detection Systems using UML, OCL, and Model-Driven Engineering (MDE).

---

## Overview

FraudDetection is a research-oriented UML profile designed to model and validate AI-based financial fraud detection systems. The project combines:

- UML modeling
- Object Constraint Language (OCL)
- Model-Driven Engineering (MDE)
- AI and Machine Learning concepts
- Formal validation using the USE tool

The profile introduces domain-specific stereotypes for representing fraud detection pipelines, machine learning models, alerts, preprocessing workflows, and quality-of-service metrics.

---

## Features

- Custom UML Profile for fraud detection systems
- AI/ML-oriented stereotypes
- Structural and behavioral OCL constraints
- USE tool validation support
- Banking fraud detection case study
- StarUML profile implementation
- Real-time fraud workflow modeling

---

## Main Stereotypes

| Stereotype | Description |
|---|---|
| `«Transaction»` | Represents a financial transaction |
| `«Customer»` | Represents a customer profile |
| `«FraudModel»` | Base stereotype for AI fraud models |
| `«Supervised»` | Supervised learning model |
| `«Unsupervised»` | Unsupervised anomaly detection model |
| `«OnlineLearning»` | Adaptive learning model |
| `«Alert»` | Fraud alert generated by the system |
| `«Preprocessing»` | Data preprocessing pipeline |
| `«QualityOfService»` | Performance and latency metrics |

---

## Validation

The profile is validated using the UML-based Specification Environment (USE).

Validation includes:

- Structural consistency checking
- Behavioral constraint verification
- OCL invariant execution
- SOIL instance simulation

### Validation Results

- 13 OCL constraints verified
- 0 validation failures
- Average latency: 47ms
- Precision: 91.2%
- Recall: 87.6%
